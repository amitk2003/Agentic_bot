#include <bits/stdc++.h>
using namespace std;

string longestPalindromeSubseq(string s) {
    int n = s.size();

    vector<vector<int>> dp(n, vector<int>(n, 0));

    // Base case
    for (int i = 0; i < n; i++)
        dp[i][i] = 1;

    // Fill DP table
    for (int len = 2; len <= n; len++) {
        for (int i = 0; i + len - 1 < n; i++) {
            int j = i + len - 1;

            if (s[i] == s[j]) {
                if (len == 2)
                    dp[i][j] = 2;
                else
                    dp[i][j] = 2 + dp[i + 1][j - 1];
            } else {
                dp[i][j] = max(dp[i + 1][j], dp[i][j - 1]);
            }
        }
    }

    int length = dp[0][n - 1];
    string ans(length, ' ');

    int start = 0;
    int end = length - 1;

    int i = 0, j = n - 1;

    while (i <= j) {
        if (s[i] == s[j]) {
            ans[start] = s[i];
            ans[end] = s[j];
            start++;
            end--;
            i++;
            j--;
        } else if (dp[i + 1][j] >= dp[i][j - 1]) {
            i++;
        } else {
            j--;
        }
    }

    return ans;
}

    cout << "Longest Palindromic Subsequence: "

response:{**What the program does**

The program finds the *longest palindromic subsequence* (LPS) of a given string and prints that subsequence.  
For the example string `"bbbab"` the output will be:

```
Longest Palindromic Subsequence: bbb
```

---

### 1.  Overview of the algorithm

1. **Dynamic programming table `dp`**  
   `dp[i][j]` stores the length of the longest palindromic subsequence in the substring `s[i…j]`.

2. **Filling the table**  
   * Base case: a single character is a palindrome of length 1 → `dp[i][i] = 1`.  
   * For every possible substring length `len` (from 2 up to `n`), iterate over all start indices `i`.  
     * Let `j = i + len - 1` be the end index.  
     * If `s[i] == s[j]`  
       * If the substring has only two characters (`len == 2`) → `dp[i][j] = 2`.  
       * Otherwise → `dp[i][j] = 2 + dp[i+1][j-1]` (the two matching ends plus the best inside).  
     * If `s[i] != s[j]` → `dp[i][j] = max(dp[i+1][j], dp[i][j-1])` (skip one end).

3. **Reconstructing the subsequence**  
   * The length of the LPS is `dp[0][n-1]`.  
   * Create a result string `ans` of that length, initially filled with spaces.  
   * Use two pointers `i` (start) and `j` (end) to walk through the original string while simultaneously filling `ans` from both ends (`start` and `end` indices).  
   * While `i <= j`:
     * If `s[i] == s[j]`, place that character at both ends of `ans`, then move inward (`i++`, `j--`, `start++`, `end--`).  
     * If the characters differ, look at the DP values:  
       * If `dp[i+1][j] >= dp[i][j-1]`, skip `s[i]` (`i++`).  
       * Else skip `s[j]` (`j--`).  
   * When the loop finishes, `ans` contains the LPS.

4. **Main function**  
   * Calls `longestPalindromeSubseq` with `"bbbab"` and prints the result.

---

### 2.  Why it works

* **DP correctness** – The recurrence mirrors the definition of a palindrome:  
  * If the ends match, they can be part of the palindrome.  
  * If they don’t, the best palindrome must lie entirely in one of the two smaller substrings.  
  * By building up from length 1 to length n, every subproblem is solved before it’s needed.

* **Reconstruction** – The DP table tells us which side to discard when the ends differ.  
  * By following the same decisions that produced the DP values, we can rebuild one optimal subsequence.  
  * Placing characters at both ends of `ans` guarantees the result is a palindrome.

---

### 3.  Complexity

* **Time** – The DP table has `n × n` cells, each filled in O(1).  
  * Filling: `O(n²)`  
  * Reconstruction: `O(n)`  
  * Total: `O(n²)`.

* **Space** – The DP table uses `O(n²)` integers, plus the output string `O(n)`.  
  * Total: `O(n²)`.

---

### 4.  Example walk‑through (`"bbbab"`)

| Step | `i` | `j` | `s[i]` | `s[j]` | Action | `dp[i][j]` |
|------|-----|-----|--------|--------|--------|------------|
| 1    | 0   | 4   | b      | b      | match  | 5 (full LPS) |
| 2    | 1   | 3   | b      | a      | skip `j` (dp[1][2] > dp[0][3]) | 3 |
| 3    | 1   | 2   | b      | b      | match  | 3 |
| 4    | 2   | 2   | b      | b      | base   | 1 |

Reconstruction places `'b'` at both ends, then `'b'` again, finally the middle `'b'`, yielding `"bbb"`.

---

**Bottom line:**  
The code uses a classic dynamic‑programming approach to compute the length of the longest palindromic subsequence and then reconstructs one such subsequence by backtracking through the DP table.}
