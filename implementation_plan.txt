Agentic AI Multi-Modal Assistant — Gemini Implementation Plan
=============================================================

Build a deployed Agentic AI application that processes text, images, PDFs, and audio simultaneously, with autonomous planning, tool orchestration, and cross-input reasoning.

-------------------------------------------------------------
USER REVIEW REQUIRED
-------------------------------------------------------------

[IMPORTANT] Gemini API Integration:
This plan has been completely rewritten to use Google Gemini API (gemini-2.0-flash or gemini-1.5-pro) exclusively. No OpenAI keys are required.

[IMPORTANT] Custom Agent Loop (No Frameworks):
As requested, we will NOT use LangChain or LangGraph. We will implement a custom ReAct (Reason + Act) loop using Gemini's native response_schema structured outputs and function calling capabilities. This perfectly fulfills the "prove you can code" requirement.

-------------------------------------------------------------
IMPLEMENTATION DETAILS
-------------------------------------------------------------

1. Custom Orchestrator
We will define Pydantic models for the execution plan:

    class ExecutionPlan(BaseModel):
        is_ambiguous: bool
        follow_up_question: str | None
        tool_calls: list[ToolCall]
        reasoning: str

We pass this schema to Gemini using response_schema=ExecutionPlan. Gemini returns a strict JSON object that we parse and execute.

2. Tools
We will register Python functions as tools. When the planner selects a tool, our Executor loop calls the Python function and appends the result to a context window.

Available Tools:
1. fetch_youtube_transcript
2. summarize_text (1-line, 3 bullets, 5-sentence formats)
3. analyze_sentiment
4. explain_code (complexity, bugs)
5. qa_search (RAG for large docs)
6. compare_documents

3. Fulfilling Test Cases
* Test Case 1 (Audio): Extract audio, pass to Gemini Audio tool -> route to summarize_text.
* Test Case 2 (PDF + Query): Extract PDF text -> Planner routes to qa_search -> returns action items.
* Test Case 3 (Image Code): Tesseract OCR (or Gemini native vision) -> route to explain_code tool.
* Test Case 4 (PDF URL to YT): Extract PDF -> URL detected -> Planner routes to fetch_youtube_transcript -> routes to summarize_text.
* Test Case 5 (Audio + PDF Compare): Extract audio + PDF text -> Planner routes to compare_documents.
